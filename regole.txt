Regole di buona formattazione :

-Su Esempi/ si inseriscono i file da cui trarre degli esempi
-Su Relazione/ si inseriscono tutti i file relativi alla relazione
----Su Grafici/ si inseriranno i *.dia dei grafici, nel caso non si userà dia allora potrebbe essere inutile
----Su Immagini/ si inseriranno le immagini presenti nella relazione, suddivise nelle giuste nelle giuste cartelle
-Nuove cartelle verrano create per il sito e il database quando diventeranno necessarie




Nomenclatura per le immagini della relazione:
i file delle immagini avranno un nome formato da :
           "a"+"b"+"c".tipofile
-"a" rappresenta a cosa appartiene l'immagine:
---"Con" per le immagini della progettazione concettuale
---"Log" per le immagini della progettazione logica, che non so ancora quante saranno

-"b" rappresenta cosa mostra l'immagine : sarà il nome dell'entità o della relazione

-"c" rappresenta il numero dell'immagine, con 2 cifre decimali : il numero "01" verrà dato alla prima immagine, e successive immagini della stessa entità o relazione, dovute a modifiche o raffinamenti, avranno un numero incrementato di 1

esempio "ConUtente03.png" è un'immagine della progettazione concettuale di utente, la terza mostrata nella relazione/creata


Regole sulle immagini della relazione:
-Sulla relazione userò "%immagine" seguito dal nome del file come ancora di dove andrà inserita quell'immagine, una volta creata
-Le immagini dovranno rappresentare quello descritto sopra dall'ultima immagine inserita e basta
---Solo nel caso di una relazione si rappresenteranno anche le 2 entità, ma per velocizzare i tempi si possono ommettere gli attributi dopo averli già rappresentati una volta, idem per le relazioni
